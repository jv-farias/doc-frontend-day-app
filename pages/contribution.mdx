## 1° Faça um fork no projeto.
 - No canto superior direito da página está o botão escrito "Fork".
 - <p className={style.textLink}><Link href="https://github.com/jv-farias/frontend-day-app" target="_blank">Nosso Projeto</Link></p>

## 2° Clone o repositório na sua maquina ou use um serviço em nuvem.
 - Use o comando abaixo para o clone do repositório na maquina local.

```git
git clone https://github.com/jv-farias/frontend-day-app.git
```
 - No caso de alguns serviço em nuvem basta copiar e colar a URL do projeto.

## 3° Crie uma branch
 - Use o comando abaixo para criar uma branch e trabalhar nas suas alterações.

```git
git checkout -b minha-branch
```

## 4° Trabalhe nas suas alterações
 - Faça as modificações que você ache necessario para a contribuição do projeto.

## 5° Faça um commit das alterações
 - Após fazer todas as alterações, faça um commit das suas alterações com uma mensagem.

```git
git add .

git commit -m "Adicione sua mensagem de commit"
```

## 6° Envie suas alterações para o seu fork
 - Mande as alterações para o seu fork usando o comando abaixo.

```git
git push origin minha-branch
```

## 7° Crie uma Pull Request (PR)
 - Volte ao repositório original e clique no botão "New Pull Request". Isso vai abrir uma página onde você pode comparar as alterações entre o repositório original e o seu fork.

## 9° Espere alguem revisar sua PR
 - Geralmente tem alguem online para revisar sua PR. Caso passe mais de 24Hrs com a PR aberta, não hesite em contatar algum dos colaboradores para revisar sua PR.
